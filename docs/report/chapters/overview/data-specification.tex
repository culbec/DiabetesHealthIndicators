\subsection{Data Specification}
\label{sec:data-specification}

\par The input dataset consists of $100,000$ patient records characterized by features based on demographics, lifestyle habits, family history, and clinical measurements that are well-established indicators of diabetes risk. The data was generated using statistical distributions from real-world medical research, ensuring \textit{privacy preservation} while reflecting realistic health patterns. The dataset is publicly available on \href{https://www.kaggle.com/datasets/mohankrishnathalla/diabetes-health-indicators-dataset}{Kaggle}.

\par Table~\ref{tbl-input-features} presents an overview of all input features used in both learning tasks. Each feature is described by its name, data type, a brief explanation, and a corresponding domain (\textit{value range}), therefore also specifying the \textbf{preconditions} of the utilized data. As confirmed by the creators of the dataset, all values fall within realistic medical ranges, matching realistic population health patterns, being a viable option when exploring lifestyle and clinical health patterns.
